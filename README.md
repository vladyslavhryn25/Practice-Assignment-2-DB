# Practical Assignment 2 - Оптимізація запитів

База даних біржі фрилансу.

5 таблиць: клієнти, фрилансери, категорії, проєкти, платежі.

Запит знаходить топ фрилансерів за сумою виплат по завершених проєктах за останній рік.

Написав два варіанти одного і того ж запиту: неоптимізований і оптимізований — і порівняв їх через EXPLAIN ANALYZE.

Неоптимізований використовує вкладені підзапити.

Один і той самий JOIN по трьох таблицях (payments → projects → freelancers) виконується тричі: для основного результату, для пошуку MAX і для PERCENTILE.

PostgreSQL щоразу сканує весь датасет заново.

Час виконання: ~238 мс

Оптимізований використовує CTE.

Спочатку один раз фільтруємо і джойнимо дані в filtered_data, потім один раз агрегуємо в agg_data, потім через RANK() знаходимо топ.

Також додав індекси на колонки, які використовуються у WHERE і JOIN — це дозволило PostgreSQL використовувати Bitmap Index Scan замість повного сканування таблиці.

Час виконання: ~31 мс

Оптимізований запит працює в 7-8 разів швидше на тих самих даних.
